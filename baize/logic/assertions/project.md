# 项目逻辑断言

用于记录项目长期目标、核心方向和跨模块判断。

## 已确认断言

- 当前项目是一个类似荒野乱斗的项目。
- Claude Code 分析 Jira Bug 单时，必须先更新 SVN 服务器上的工程分析地址，再结合工程现状与 Jira Bug 单信息给出分析；禁止只根据 Jira Bug 单文本下结论，因为这种分析会流于表面、缺少工程依据。
- 如果 SVN 更新失败，仍必须基于当前可读取的工程工作副本继续进行工程级 BUG 分析，并在结论中明确标注 SVN 更新失败原因、当前分析依据来自未完成更新的本地工程状态；不得因为 SVN 更新失败直接跳过工程分析。
- 所有涉及 Jira BUG 单的分析、排查、根因判断、AI 分析评论生成流程，都必须优先走工程级 BUG 分析任务；只有用户明确提供固定评论内容且不要求分析时，才允许走普通 Jira 评论写入流程。
