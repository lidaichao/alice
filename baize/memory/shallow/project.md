## 项目概述

这是一个使用 Go 语言编写的多人游戏服务器项目，采用 SVN 版本控制，基于自定义 GOPATH 模式管理依赖。项目采用模块化架构，通过统一入口 `main.go` 启动不同类型的服务器。

## 构建命令

### 环境设置
项目使用 GOPATH 模式，需要设置环境变量：
```bash
# Windows
SET GOPATH=/path/to/union/deps:/path/to/union

# Linux/Mac
export GOPATH=/path/to/union/deps:/path/to/union
```


### 构建
```bash
# Linux 构建（项目根目录）
./build.sh

# Windows 构建
cd buildEnv && buildLinuxUnion.bat   # Linux 版本
cd buildEnv && WindowsEnv.bat        # Windows 环境

# 手动构建
go build -o union main.go

# 跨平台构建（Linux 目标）
SET CGO_ENABLED=0
SET GOOS=linux
SET GOARCH=amd64
go build -o union main.go
```

### 运行
```bash
# 基本用法
go run main.go -f conf/<config_file>.json

# 常用示例
go run main.go -f conf/logic_1001.json      # 逻辑服务器
go run main.go -f conf/pvp_4001.json        # PvP 服务器
go run main.go -f conf/auth_11.json         # 认证服务器

# 运行参数
# -f: 配置文件路径（必需）
# -d: 守护进程模式（1 启用）
# -n: 局域网模式（1 启用）
```

```bash
# Linux/Mac
```

## 概述

KPS（Key Performance Score）是PVP战斗结算后对每个玩家的综合表现评分，用于评选MVP和展示玩家评分。

评分由5部分组成：基础数据分、失误防守分、团队表现分、对位表现分、高光表现分。

## 最终公式

```
中间MVP分 = 基础数据分 + 失误防守分 + 团队表现分 + 对位表现分 + 高光表现分
最终MVP分 = max(最终MVP分最小值, 6 * ln(中间MVP分 + 1) - 5)
```

最终MVP分存储到 `GamerGeneralData.Kps`（乘以10000转int32）。

### 各部分分数最大值限制

从 `normal_const_lua` 配置表读取，配置值已乘10000：

| 字段 | 含义 | 默认值 |
|------|------|--------|
| stats_score_max | 基础数据分最大值 | 500000 (50.0) |
| turnover_defense_score_max | 失误防守分最大值 | 50000 (5.0) |
| team_performance_score_max | 团队表现分最大值 | 250000 (25.0) |
| matchup_score_max | 对位表现分最大值 | 50000 (5.0) |
| highlight_score_max | 高光表现分最大值 | 50000 (5.0) |
| final_mvp_score_min | 最终MVP分最小值 | 10000 (1.0) |

各部分分数计算后，若超过最大值则截断；失误防守分可正可负（正数表示防守好失误少，负数表示防守差失误多），只限制正最大值，负数不限制。

---

## 第一部分：基础数据分

```
基础数据分 = 得分 * 得分系数
           + 篮板 * (篮板系数 + 篮板动态系数)
           + 盖帽 * (盖帽系数 + 盖帽动态系数)
           + 抢断 * 抢断系数
           + 助攻 * 助攻系数
           + 地板球 * 地板球系数
```

- 抢断 = 抢断成功数 + 截球数
- 篮板动态系数：从 `mvp_rating_parameter` 表按本场双方篮板总数查找 `rebound_dynamic_param`，超出最大行取最大行值
- 盖帽动态系数：从 `mvp_rating_parameter` 表按本场双方盖帽总数查找 `block_dynamic_param`，超出最大行取最大行值

## 第二部分：失误防守分

```
失误防守分 = -(两分未命中 * 两分未命中系数
             + 三分未命中 * 三分未命中系数
             + 被盖帽数 * 被盖帽系数
             + 被干扰数 * 被干扰系数
             + 被严重干扰数 * 被严重干扰系数
             + 被撞停次数 * 被撞停系数
             + 被截球次数 * 被截球系数
             - 干扰数 * 干扰系数
             - 严重干扰数 * 严重干扰系数)
```

- 两分未命中 = (TwoN-TwoG) + (DunkN-DunkG) + (LayupN-LayupG)，包含中投、灌篮、上篮
- 三分未命中 = ThreeN - ThreeG
- 干扰数 = disturbN - seriousDisturbN（总干扰减去严重干扰）
- 干扰和严重干扰是防守正向贡献，会抵消扣分

## 第三部分：团队表现分

```
团队表现分 = 加速次数 * 加速系数
           + 挡拆成功次数 * 挡拆系数
           + 晃倒次数 * 晃倒系数
           + 持球分 * 持球占比系数 * 得分 * 得分系数
           + 篮板 * 篮板系数 * 篮板占比 * 篮板占比系数
           + 盖帽 * 盖帽系数 * 盖帽占比 * 盖帽占比系数
           + 助攻 * 助攻系数 * 助攻占比 * 助攻占比系数
```

### 持球分计算

```
持球占比 = 玩家持球时间 / 友方所有玩家持球时间总和    （范围 [0, 1]）
持球分 = max(0, (3 - (29/6) * 持球占比 + (5/3) * 持球占比^2) / 持球占比参数)
```

- 持球占比参数（id=17）按球员位置区分（PG/SG/SF/PF/C），不匹配时默认取C
- 通过 `GetPlayerPosByPlayerId` 获取球员位置（100=PG, 200=SG, 300=SF, 400=PF, 500=C）

### 占比计算

- 篮板占比 = 玩家篮板 / 友方篮板总和，范围 [0, 1]
- 盖帽占比 = 玩家盖帽 / 友方盖帽总和，范围 [0, 1]
- 助攻占比 = 玩家助攻 / 友方助攻总和，范围 [0, 1]

## 第四部分：对位表现分

```
对位表现分 = (玩家篮板 - 对位玩家篮板) * 对位篮板系数
           + (玩家盖帽 - 对位玩家盖帽) * 对位盖帽系数
```

- 对位玩家通过 `PlayerPvpResultData.oppositeGid` 字段指定
- 对位差值可以为负数（对面更强时扣分）
- 对位玩家数据从对方队伍的 `GamerGeneralData` 中获取

## 第五部分：高光表现分

```
高光表现分 = 绝杀次数 * 绝杀系数
```

- 绝杀次数来自 `PlayerPvpResultData.finalHit`
