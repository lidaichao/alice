# Alice 问题评测 — 协调者说明（非技术版）

您**不用懂编程**。只需改一个记事本文件，其余由电脑自动完成。

---

## 您要做的只有两件事

### 1. 录入问题

1. 用 **记事本** 打开文件（在项目文件夹里）：

   `eval\data\coordinator_questions.txt`

2. **一行写一个问题**，例如：

   ```
   CT-10888 今天程序提交了什么
   本周需要完成的任务有哪些？
   ```

3. 以 `#` 开头的行是说明，可以不管；空行会自动忽略。

4. **保存** 文件。

（可选）若一行里用 **Tab 键** 分两段：左边是问题，右边是您心里的「期望答案」，系统会记作标准；**不填也可以**。

### 2. 运行评测

任选一种：

- **双击** 项目根目录下的 `run_coordinator_eval.bat`
- 或在 Cursor 里对助手说：**「跑问题评测」**

---

## 看结果

打开报告（用记事本或浏览器均可）：

`eval\reports\协调者报告_latest.md`

只看三样：

| 列 | 含义 |
|----|------|
| **通过** | Alice 表现符合自动检查 |
| **未通过** | 需要看「说明」列里的白话原因 |
| **未测** | 通常是 Alice 服务没开，见下文 |

---

## 第一次使用前（只需配一次）

1. **启动 Alice 服务**  
   请技术人员帮您在本机运行 `backend\ai_bridge.py`，或您有桌面版 Alice 且已连上同一后台。

2. **Jira 权限**  
   在 Admin 后台或本机配置里填好 Jira 令牌（PAT），否则涉及 Jira/提交的问题会测不准。

服务没开时，仍可运行 bat，但报告里会写「未测 / 服务未连接」。

---

## 未通过时怎么办

| 情况 | 您做什么 |
|------|----------|
| 确认是 Alice 答错了 | 把报告里「第几题 + 说明」发给开发，或在 Cursor 说「第 2 题 Alice 错了请修」 |
| 题目或期望变了（数据过期） | 改 `coordinator_questions.txt` 里该行，或 Tab 后补上期望说明 |
| 显示「未测 / 服务未连接」 | 先启动 Alice 服务，再双击 bat 重跑 |

您**不需要**打开 CSV、不需要记命令、不需要在聊天里把问题问几十遍。

| 文件 | 您是否编辑 |
|------|------------|

---

## 常见问题

**问：可以加多少题？**  
答：建议先 10～30 条代表日常场景，不必几百条。

**问：要不要每天跑？**  
答：Alice 或后台有更新时跑一遍即可；发版前跑一遍更稳妥。

**问：和灰盒手册关系？**  
答：手册里手测的 5 条，您也可以写进 txt，由本评测自动问 Alice，减少手工重复。
