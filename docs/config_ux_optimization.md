# 设置面板 UX 优化方案：面向非技术用户

> 问题：当前配置项（22项）大量技术参数对项目经理用户不可读
> 方案：参考 ChatGPT / Claude / WorkBuddy，极简化 + 术语翻译 + 隐藏高级

---

## 一、竞品配置策略对比

| 产品 | 暴露的配置项 | 技术参数 | 策略 |
|------|------------|---------|------|
| **ChatGPT** | 账号、主题、语言、自定义指令 | **0个** | 全隐藏，模型选择在聊天头 |
| **Claude** | 账号、外观、Beta功能 | **0个** | 全隐藏，模型选择在聊天头 |
| **WorkBuddy** | 连接器向导、插件市场、模型选择 | **0个** | 向导式，技术细节黑盒 |
| **Cursor** | 模型选择、快捷键、主题 | **0个** | 开发者工具，但不暴露API参数 |
| **Copilot** | 账号授权 | **0个** | 最简，一键连接 |

**结论**：主流产品对终端用户暴露 **0 个技术参数**。所有 API Key、模型参数、缓存策略都对用户不可见。

## 二、当前问题诊断

### 问题分类

| 类型 | 示例 | 问题 |
|------|------|------|
| **黑话/术语** | max_tokens, temperature, L1_CHARS, cache TTL | 项目经理完全看不懂 |
| **JSON 输入** | `{"jira":500,"svn":600,...}` | 需要编程知识 |
| **无意义数字** | 8000, 300, 120, 0.7 | 不知道改大改小的后果 |
| **内部概念** | Bridge端口, 上下文上限, 语义缓存 | 这是开发者才关心的 |
| **安全问题** | API Key 明文输入框，密码可见切换 | 不够安全 |

## 三、优化方案：三级信息架构

### Level 1：快速开始（默认，3个区域）

| 区域 | 配置项 | 交互方式 |
|------|--------|---------|
| **Jira 连接** | 地址 + 账号 + 密码/PAT | 文本输入 + 测试按钮 |
| **AI 模型** | 模型选择 | 下拉菜单（带推荐标签） |
| **知识库** | Notion/GDrive/SVN | 可折叠，+ 按钮展开向导 |

### Level 2：偏好设置（展开后，2个滑块）

| 配置项 | 原始参数 | 新交互 | 说明 |
|--------|---------|--------|------|
| 回答风格 | temperature | 滑块：精确 ↔ 创意 | "需要精确数据靠左，需要创意靠右" |
| 回答长度 | max_tokens | 滑块：简洁 ↔ 详细 | "简短摘要靠左，完整分析靠右" |

### Level 3：开发者选项（默认隐藏，需手动开启）

| 配置项 | 原始参数 | 触发条件 |
|--------|---------|---------|
| API 地址 | deepseek_url | 高级开关打开后 |
| 上下文阈值 | context_limit, L1_CHARS | 同上 |
| 缓存设置 | cache_ttl | 同上 |

## 四、术语翻译对照表

| 原始术语 | 面向用户表述 | 说明文案 |
|---------|------------|---------|
| `jira_url` | Jira 服务器地址 | "公司 Jira 的网址，如 http://jira.公司.com:8080" |
| `jira_pat` | 访问令牌 / 密码 | "Jira 登录密码，或个人访问令牌（PAT）" |
| `model_name` | AI 模型 | 下拉：[DeepSeek-V3 推荐] [DeepSeek-R1] |
| `max_tokens` | 回答长度 | 滑块：简洁 ←→ 详细 |
| `temperature` | 回答风格 | 滑块：精确 ←→ 创意 |
| `deepseek_url` | (隐藏) API 地址 | 高级模式才显示 |
| `bridge_port` | (隐藏) 服务端口 | 开发者选项 |
| `context_limit` | (隐藏) 检索范围 | 开发者选项 |
| `cache_ttl` | (隐藏) 缓存时间 | 开发者选项 |
| `L1_CHARS` | (隐藏) 数据源配额 | 开发者选项 |
| `notion_key` | Notion 连接令牌 | 向导式："打开 Notion → 设置 → 集成 → 创建令牌" |
| `gdrive_key` | Google Drive API 密钥 | 向导式："打开 Google Cloud → API → 创建密钥" |
| `svn_url` | SVN 仓库地址 | "代码仓库地址，如 https://svn.公司.com/proj" |
| `svn_user/pass` | SVN 账号密码 | "SVN 登录凭据" |

## 六、实施建议

### Phase 1：立即执行（用户体验最大提升）

1. 将技术参数全部折叠到"开发者选项"（默认隐藏）
2. 模型选择改为下拉菜单（预设 DeepSeek-V3 / DeepSeek-R1）
3. 知识库改为向导式连接（逐步引导 + 说明文案）
4. 添加"测试连接"按钮 + 状态指示灯

### Phase 2：体验增强

5. temperature → "回答风格"滑块
6. max_tokens → "回答长度"滑块
7. 首次启动引导流程（3步向导）
8. 错误信息翻译为用户语言

### Phase 3：极致体验

9. Jira 项目列表自动拉取（输入地址后自动获取）
10. 知识库连接状态持久化显示
11. 连接失败时的友好恢复建议
