# 爱丽丝 桌面客户端 UX 布局优化方案

> 基于 WorkBuddy / Claude Code Desktop / ChatGPT 等主流 AI 桌面端 UX 分析
> 目标：为 Jira 项目管理场景打造高效、专业的桌面 AI 助手体验

## 二、爱丽丝 UX 定位

爱丽丝是 **Jira 项目管理 AI 助手**，核心场景：

| 场景 | 频率 | UX 需求 |
|------|------|---------|
| 查询任务/日报/周报 | 每日多次 | 快速输入 + 结构化结果展示 |
| Jira 创建/更新 | 频繁 | 确认卡可视化 |
| 代码审查（SVN Diff） | 按需 | 差异对比视图 |
| 文档检索（Notion/GDrive） | 按需 | 上下文来源展示 |
| 多项目切换 | 频繁 | 会话分组/项目筛选 |

## 四、核心 UX 组件详细设计

### 4.1 会话管理侧栏（左侧 240px-280px）

借鉴 Claude Code 的会话管理中心 + 白泽的多会话模式：

| 组件 | 说明 |
|------|------|
| **搜索框** | 按标题/项目搜索历史会话 |
| **新建按钮** | `+ 新建会话`，快捷键 `Ctrl+N` |
| **项目筛选** | 标签形式：全部 / CT / 其他项目... |
| **状态筛选** | 活跃 / 已归档（借鉴 Claude 自动归档） |
| **会话列表** | 显示标题 + 项目标签 + 最后活跃时间 + 消息数 |
| **右键菜单** | 重命名 / 归档 / 删除 / 导出 |
| **底部** | 设置齿轮图标 |

### 4.2 对话区（中央，弹性）

| 组件 | 说明 |
|------|------|
| **会话标题栏** | 显示当前会话名称，可点击重命名 |
| **模型选择器** | DeepSeek / 其他（下拉，参考 WorkBuddy） |
| **视图模式** | 简洁 / 标准 / 详细（借鉴 Claude 三档模式） |
| **上下游切换** | `Ctrl+Tab` / `Ctrl+Shift+Tab` |
| **消息气泡** | 用户蓝色右侧 + AI 白色左侧，支持 Markdown 渲染 |
| **确认卡** | 橙色边框卡片，确认/拒绝按钮（P0 已实现） |
| **输入区** | 多行输入 + 附件按钮 + 快捷发言按钮 |
| **意图指示器** | 输入时显示分类结果（🔍检索 / 🛡️需确认 / ⛔危险） |

### 4.3 上下文面板（右侧 280px-320px，可折叠）

借鉴 WorkBuddy 结果区 + Claude 工作区：

| 组件 | 说明 |
|------|------|
| **知识库来源** | 展示本次检索命中的数据源（Jira/SVN/Notion/GDrive） |
| **Issue 预览** | 点击 Issue Key 弹出详情卡片 |
| **操作记录** | 本次会话的操作时间线 |
| **快捷操作** | 贴为评论 / 复制 Markdown / 导出报告 |
| **代码 Diff** | SVN 文件差异对比（借鉴 Claude Diff Viewer） |

### 4.4 顶部标题栏

| 组件 | 说明 |
|------|------|
| **Logo + 产品名** | 爱丽丝 Jira AI |
| **连接状态** | 🟢已连接 / 🟡启动中 / 🔴离线 |
| **当前项目** | 下拉选择活跃项目 |
| **窗口控制** | 最小化 / 最大化 / 关闭（自定义标题栏） |

## 五、交互模式

### 5.1 快捷键设计

| 快捷键 | 功能 | 借鉴来源 |
|--------|------|----------|
| `Ctrl+N` | 新建会话 | Claude Code |
| `Ctrl+Tab` | 下一个会话 | Claude Code |
| `Ctrl+Shift+Tab` | 上一个会话 | Claude Code |
| `Ctrl+B` | 切换侧栏 | WorkBuddy |
| `Ctrl+R` | 切换上下文面板 | WorkBuddy |
| `Ctrl+Enter` | 发送消息 | 通用 |
| `Ctrl+K` | 快速搜索会话 | 通用 |
| `Ctrl+,` | 打开设置 | 通用 |
| `Alt+Space` | 全局唤起（迷你窗口） | ChatGPT |
| `Escape` | 关闭弹窗/取消流式 | 通用 |

### 5.2 确认卡交互（P0 已实现，UX 增强）

```
AI生成确认卡
  ↓
橙色确认卡出现在对话流中
  ├── 展示：问题类型 | 标题 | 项目 | 经办人
  ├── 警告：缺少字段 / 标签不支持
  ├── [✅ 确认创建] [✕ 取消]
  ↓ 用户确认
状态动画：⏳ 正在创建... → ✅ 已创建 CT-9999
  ↓ 失败时
自动切换为恢复卡片：[移除标签后重试] [取消]
```

### 5.3 多项目上下文切换

```
顶部项目选择器: [CT 足球小将 ▼]
  ├── CT 足球小将     ← Jira: ctjira1:8080
  ├── DT 新项目       ← Jira: dtjira:8080
  └── + 添加项目...

切换项目时：
  → 侧栏会话列表按项目重新筛选
  → 上下文面板切换到新项目数据
  → AI Bridge 自动切换 Jira 实例
```
