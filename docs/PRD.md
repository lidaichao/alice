## 一、产品概述

### 1.1 产品定位

**爱丽丝** 是一款面向 Jira 项目管理者的 AI 桌面助手。用户无需打开 Jira Web 界面，在桌面客户端中通过自然语言即可：

- 查询任务进度、生成日报/周报
- 创建和更新 Jira Issue
- 检索代码仓库（SVN）变更记录
- 搜索团队文档（Notion / Google Drive）
- 获得 AI 驱动的项目管理建议

### 1.2 核心价值

| 痛点 | 解决方案 |
|------|---------|
| 每次查任务都要打开浏览器登录 Jira | 桌面端一键启动，本地持久登录 |
| 跨 Jira/SVN/文档查找信息效率低 | AI 自动并发检索 4 个数据源 |
| 手动整理日报/周报耗时 | 自然语言一句话生成结构化报告 |
| 非技术人员配置复杂 | 极简设置面板，3 步上手 |

### 1.3 目标用户

| 角色 | 场景 | 频率 |
|------|------|------|
| 项目经理 | 查看团队任务进度、生成周报 | 每日 |
| 技术主管 | 代码审查、SVN 变更追踪 | 按需 |
| 策划/产品 | 查询需求文档、创建 Issue | 频繁 |

## 三、用户流程

### 3.1 首次使用

```
双击 爱丽丝.exe
  → 自动检测 Python 环境
  → 弹出设置面板
      ├── 输入 Jira 地址 + 账号密码 → 测试连接 ✅
      ├── 选择 AI 模型（DeepSeek-V3 推荐）
      └── 可选：连接 Notion / GDrive / SVN
  → 保存 → 进入主界面
```

### 3.2 日常使用

```
打开爱丽丝
  → 点击「+ 新建」创建会话
  → 输入问题：「帮我查这周 CT 项目未完成的 Bug」
  → AI 自动检索 Jira + 流式输出结果
  → 追问：「把 CT-1001 指派给张三」
  → 弹出确认卡 → 点击确认 → 完成
```

## 四、非功能需求

| 维度 | 要求 |
|------|------|
| 启动速度 | 首次 < 30s，后续 < 5s |
| AI 首字延迟 | < 1s（SSE 流式） |
| 离线可用性 | 桌面端离线仍可查看历史会话 |
| 数据安全 | 密码本地加密存储，API Key 不传输到第三方 |
| 兼容性 | Windows 10/11 |
| 安装包大小 | < 300MB（含 Electron + Python 运行时） |
